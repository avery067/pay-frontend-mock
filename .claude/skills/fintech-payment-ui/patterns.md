# 支付域 UX 模式库（结汇 / 发卡 / 收单）

配合 `SKILL.md` 使用。所有颜色走 `themes.css` 的语义 token，不要硬编码红绿。

## 通用：金额与状态

### 金额显示
- **等宽拉丁数字**：加 `.amount`（`tabular-nums`），保证列表/表格纵向对齐。
- **千分位分组**：`1,234,567.89`；超大额可用窄空格分组 `1 234 567.89`。
- **货币**：代码固定放同一侧（推荐 `USD 1,234.56` 或 `1,234.56 USD`，全站统一）；符号与代码二选一别混用。
- **正负**：同时用**符号 + 颜色**——`+1,200.00`（`--pos`）/ `−350.00`（`--neg`，用真减号 `−` U+2212 不是 hyphen）。**永不只靠红绿**（色盲不可读）。
- **精度**：法币 2 位；加密/汇率可 6–8 位，超出部分弱化字重或灰度。
- **脱敏/隐藏**：前台余额可一键 `••••••` 切换。

### 状态语义（Badge + 文字，勿只用色点）
| 语义 | token | 典型状态词（中 / EN） |
|---|---|---|
| 成功/终态 | `--success` | 已结算 Settled · 已到账 Paid out · 成功 Succeeded |
| 处理中/待处理 | `--warning` | 处理中 Processing · 待审核 In review · 待结算 Pending |
| 失败/拒绝 | `--danger` | 失败 Failed · 被拒 Declined · 已退回 Returned |
| 信息/中性提示 | `--info` | 需补件 Action needed |
| 中性/关闭 | `--muted-foreground` | 草稿 Draft · 已取消 Canceled · 已关闭 Closed |

关键状态 Badge 后跟**可点击的下一步**（Wise 式"每条都可操作"）：`待审核 → 去补件`、`失败 → 查看原因`。

### 空 / 错 / 载
- **空态**是行动邀请：一句话说清 + 一个主 CTA（"还没有卡片 · 申请第一张虚拟卡"）。
- **错误**说清哪错了、怎么修，用界面口吻，不道歉不含糊。给错误码时附可复制/可查文档链接。
- **加载**用骨架屏（skeleton）匹配真实布局，别用居中转圈；金额位留等宽占位防跳动。

## 结汇（FX / Settlement）— 偏 Wise 舒适

**核心心理**：用户最焦虑的是"发起到确认之间"的空窗（Wise 洞察）。所以**先给状态与到手金额，再给余额**。

- **报价卡（Quote）**——全透明，是结汇的门面：
  ```
  ┌─────────────────────────────────────┐
  │ 你支付        USD  10,000.00   ⌄     │
  │ ───────────────────────────────────  │
  │ 汇率 Rate     1 USD = 7.1820 CNY  🔒  │  ← 锁汇倒计时 00:28
  │ 点差 Spread   0.35%          −25.14   │  ← 逐项列清，别藏
  │ 手续费 Fee    USD 12.00               │
  │ 预计到账      1–2 个工作日             │
  │ ═══════════════════════════════════  │
  │ 对方收到      CNY  71,676.86          │  ← 最终到手，最大最粗
  └─────────────────────────────────────┘
  ```
- **金额输入**：大号可编辑数字 + 币种下拉（带旗标/币种代码），支持"我付 / 对方收"双向联动。
- **锁汇倒计时**：`--warning` 进度条 + `mm:ss`，到期给"刷新汇率"而非静默失效。
- **结算时间线**：发起 → 合规审核 → 兑换 → 汇出 → 已到账；每步带时间戳，当前步高亮，失败步给原因与下一步。
- **合规/KYC 闸口**：金额/国家触发时，用 `--info` 提示"需补充资料"，内联表单而非跳走；说清为什么要（反洗钱），别用恐吓文案。
- **回执 Receipt**：可下载/分享，含参考号（等宽可复制）、各项费用、汇率、时间戳。

## 发卡（Card Issuing）— 后台偏 Mercury，前台偏 Revolut

- **卡片视觉组件**：`aspect-ratio: 1.586`（信用卡比例 85.6×54mm），可翻面看 CVV；卡号 `•••• •••• •••• 4242` 等宽脱敏；卡种/网络 logo；虚拟卡加轻微渐变或品牌色，实体卡更克制。
- **卡片列表**：网格（前台）或表格（后台）；每卡显示别名、末四位、状态、本月已用/额度进度条。
- **开卡流程**：类型（虚拟/实体）→ 用途/命名 → 额度与规则 → 确认。分步进度可见，可返回改。
- **消费控制**：单笔/日/月额度、商户类目白/黑名单、地区限制、单次授权开关；改动即时预览影响。
- **冻结/注销**：冻结可逆（`--warning`），注销不可逆（`--danger` + 二次确认，输入末四位确认）。
- **卡交易流水**：商户名 + 图标、时间、金额（`--neg` 支出）、授权/清算状态、可展开看授权详情。

## 收单（Acquiring）— 偏 Mercury·Ramp 高密度

- **商户仪表盘**：顶部 4 张 KPI（今日交易额、笔数、成功率、待结算），Role-Metric 原则——把商户最关心的一个数放最左最大。
- **交易表格**：高密度、`tabular-nums` 对齐、可粘性表头、列可排序/筛选（时间、状态、金额区间、支付方式、订单号）。行内状态 Badge，行可展开看时间线。批量操作放表头。
- **对账 / 清算（Reconciliation / Settlement）**：按结算批次（payout batch）聚合，展示批次总额 = Σ交易 − 手续费 − 退款；差异（discrepancy）行用 `--warning`/`--danger` 高亮并给"去核对"。
- **退款 / 争议（Refund / Dispute / Chargeback）**：退款金额校验（≤可退），争议给举证倒计时（`--warning`）与材料上传。
- **账期/清算周期**：可视化下一次打款时间与金额（T+1 / T+7…），待结算与已结算分区。
- **支付方式 / 通道**：各卡组织/本地钱包的开通状态、费率、成功率对比。

## 信任线索速查（做对，别做成 slop）
- ✅ 费率/汇率/到账时间前置且逐项透明；✅ 每笔可追溯、可复算、给真实下一步；✅ 敏感信息脱敏；✅ KYC/限额用真实信息提示。
- ❌ `🔒 SECURE OPERATOR AUTHENTICATION` 这类填充式"安全"文案；❌ 编造的实时遥测/假徽章；❌ 把标准动作换成主题化黑话（`Authenticate Session` 代替 `继续`）。
